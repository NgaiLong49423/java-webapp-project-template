# Glossary - Giải thích thuật ngữ

## Agent

Agent là AI có nhiệm vụ cụ thể, có thể đọc đầu vào, xử lý theo quy tắc và sinh đầu ra theo format đã định.

## Skill

Skill là một khả năng riêng của Agent. Ví dụ: tách FR, tạo Issue, tạo Milestone.

## Orchestrator

Orchestrator là bộ điều phối. Nó không làm hết mọi việc mà quyết định gọi skill nào, theo thứ tự nào.

## SRS

SRS là Software Requirements Specification, tức tài liệu đặc tả yêu cầu phần mềm. Nó mô tả hệ thống cần làm gì.

## FR

FR là Functional Requirement, tức yêu cầu chức năng. Ví dụ: hệ thống cho phép khách đặt lịch rửa xe.

## NFR

NFR là Non-Functional Requirement, tức yêu cầu phi chức năng. Ví dụ: bảo mật, hiệu năng, dễ dùng.

## Epic

Epic là nhóm chức năng lớn. Ví dụ: Booking Management.

## User Story

User Story là câu chuyện người dùng, thường viết theo mẫu: As a..., I want..., so that...

## Task

Task là việc cụ thể để một người có thể làm. Ví dụ: tạo API đặt lịch.

## Layer

Layer là lớp kỹ thuật. Ví dụ: Database, Backend, Frontend, Testing.

## Label

Label là nhãn trong GitHub Issue, dùng để phân loại công việc.

## Milestone

Milestone là mốc hoàn thành lớn. Ví dụ: Sprint 1 - Core Booking Flow.

## Story Point

Story Point là điểm ước lượng độ khó của task. Điểm càng cao thì task càng khó hoặc tốn thời gian.

## Acceptance Criteria

Acceptance Criteria là tiêu chí nghiệm thu. Nó trả lời câu hỏi: làm xong thế nào thì được tính là hoàn thành?

## Definition of Done

Definition of Done là định nghĩa hoàn thành. Nó gồm code, test, review, tài liệu, và không còn lỗi nghiêm trọng.

## RBAC

RBAC là Role-Based Access Control, tức phân quyền theo vai trò. Ví dụ: Admin được xóa, Customer chỉ được xem dữ liệu của mình.

## CRUD

CRUD là Create, Read, Update, Delete. Nghĩa là tạo, xem, sửa, xóa dữ liệu.

## API Contract

API Contract là bản mô tả rõ endpoint, request, response, lỗi và quyền truy cập của API.

## CLI

CLI là Command Line Interface, tức giao diện dòng lệnh. GitHub CLI là công cụ `gh` để thao tác GitHub bằng lệnh.
